# Misconception Graph

A running index of the misconception graph described in `blueprint.md`
§16. Each row mirrors a "Common Misconceptions" block from a chapter —
this table exists so a later chapter never re-introduces the same
misconception under a different analogy without noticing. The `ID` column
is the stable key referenced by `concept-graph.yaml`'s `misconception_ids`.

Seeded from blueprint §16's registry of 38 durable misconceptions; the
`Analogy` and full `Correct Intuition` phrasing are filled in when each
chapter is actually drafted (blueprint §11 gives the full per-chapter
misconception list, of which this registry tracks one representative
entry per chapter).

| `bandwidth-equals-latency` | Bandwidth vs. latency | Bandwidth and latency are the same. | Bandwidth is capacity; latency is delay — a link can be wide and slow, or narrow and fast. | Flashlight signals across a field | Ch. 2 |

| `switch-router-same-job` | Switches vs. routers | A switch and a router perform the same job. | A switch forwards frames within one local network using link-layer addresses; a router forwards packets between networks using IP addresses. | Building mailroom and corridors | Ch. 4 |
| `ip-permanently-identifies-machine` | IP addresses | An IP address permanently identifies one machine. | An IP address identifies an interface's position within a routing hierarchy at a point in time — the same device can be reassigned a new address, share one address via translation, or hold different addresses on different interfaces. | Hierarchical postal address | Ch. 6 |
