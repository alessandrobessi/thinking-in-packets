# Bibliography

Master citation list, aggregated from the per-chapter files in this
directory. Per `blueprint.md` §19, technical claims should cite:

- **IETF RFCs and working-group documents** for Internet protocols;
- **IEEE standards or authoritative summaries** for Ethernet and Wi-Fi;
- **official browser, OS, cloud-platform, and Kubernetes documentation**
  for implementation behavior;
- **peer-reviewed or original research** for congestion control, routing
  incidents, and measurement claims;
- **original historical documents** for the development of major
  protocols.

Source discipline: historical anecdotes require a source; performance
numbers require context, date, environment, and methodology; statements
about current protocol adoption must be dated; vendor documentation
establishes how that vendor implements a feature, not a universal
networking law; security claims must name the threat model.

This file is a flat, deduplicated index; see `references/chapter-NN.md`
for which chapter cites which source.
